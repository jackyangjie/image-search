# AI 相册搜索 APP - 产品需求文档 (PRD)

## 1. 产品概述

### 1.1 产品名称
AI 相册搜索 / SmartPhoto Search

### 1.2 产品定位
一款基于自然语言描述的本地智能相册搜索工具，让用户通过描述性文字快速找到手机中的照片。

### 1.3 核心价值主张
- **解决痛点**：手机照片成千上万，传统相册难以快速定位特定照片
- **隐私优先**：所有AI处理本地完成，照片不上传云端
- **极致体验**：自然语言描述，毫秒级搜索结果

### 1.4 目标用户
| 用户群体 | 特征 | 使用场景 |
|---------|------|---------|
| 摄影爱好者 | 照片数量庞大（1万+），需要按场景查找 | 找"去年秋天银杏大道的照片" |
| 宝妈/宝爸 | 大量孩子照片，需要按时间/场景/表情查找 | 找"宝宝第一次走路的照片" |
| 商务人士 | 截图、文档照片多，需要快速检索 | 找"上周的会议白板照片" |
| 隐私敏感用户 | 拒绝云同步，要求本地处理 | 所有个人照片管理 |

## 2. 功能需求

### 2.1 核心功能

#### 2.1.1 自然语言搜索 (P0)
- **描述**：用户输入自然语言描述，返回语义匹配的照片
- **示例**：
  - "海边的日落" → 找到海边日落照片
  - "穿红衣服的小明在公园" → 找到对应场景照片
  - "去年春节全家福" → 按时间+内容筛选
- **输入限制**：最多100个字符
- **响应时间**：< 500ms（本地索引）

#### 2.1.2 相册扫描与索引 (P0)
- **描述**：自动扫描本地相册，提取AI向量建立索引
- **功能点**：
  - 首次启动自动扫描
  - 后台增量更新（新增照片自动索引）
  - 显示扫描进度和统计
  - 支持暂停/恢复扫描
- **性能要求**：
  - 索引速度：> 10张/秒（中端手机）
  - 存储占用：原始照片的5-10%

#### 2.1.3 照片浏览 (P0)
- **描述**：网格视图浏览所有已索引照片
- **功能点**：
  - 时间倒序排列
  - 缩略图展示
  - 下拉刷新
  - 点击查看大图
  - 显示索引状态（已索引/待索引）

### 2.2 辅助功能

#### 2.2.1 搜索建议 (P1)
- 热门搜索词推荐
- 搜索历史记录
- 智能补全

#### 2.2.2 结果筛选 (P1)
- 按时间范围筛选（今天/本周/本月/自定义）
- 按相似度阈值调整

#### 2.2.3 相似图片搜索 (P2)
- 选择一张照片，找到相似的照片
- 基于向量相似度排序

### 2.3 未来功能 (P3)
- 人脸识别与分组
- 地理位置聚类
- 智能相册自动生成
- 云端备份（可选）

## 8. 数据埋点设计

### 8.1 埋点原则

- **隐私优先**: 所有埋点数据匿名化，不包含用户标识
- **本地优先**: 埋点数据先存储本地，可选上传
- **最小化**: 只收集产品优化必需的数据
- **透明**: 用户可查看并关闭数据收集

### 8.2 埋点事件列表

#### 8.2.1 业务事件

| 事件名称 | 触发时机 | 参数 | 用途 |
|---------|---------|------|------|
| app_launch | 应用启动 | {cold_start: boolean} | 统计DAU、启动类型 |
| search_performed | 执行搜索 | {query_length, has_results, result_count} | 搜索功能使用分析 |
| search_result_clicked | 点击搜索结果 | {position, similarity_score} | 搜索结果质量评估 |
| scan_started | 开始扫描 | {total_photos} | 扫描功能使用 |
| scan_completed | 扫描完成 | {duration, indexed_count, failed_count} | 扫描性能分析 |
| photo_viewed | 查看照片 | {from_search: boolean} | 用户行为分析 |
| permission_granted | 授权成功 | {permission_type} | 权限转化率 |
| permission_denied | 授权拒绝 | {permission_type} | 权限问题分析 |

#### 8.2.2 性能事件

| 事件名称 | 触发时机 | 参数 | 用途 |
|---------|---------|------|------|
| search_latency | 搜索完成 | {duration_ms, query_type} | 性能监控 |
| scan_speed | 批量索引 | {photos_count, duration_ms} | 索引性能监控 |
| ai_encode_time | AI编码 | {type: 'image'/'text', duration_ms} | AI性能监控 |
| db_query_time | 数据库查询 | {operation, duration_ms} | DB性能监控 |
| memory_warning | 内存警告 | {used_mb, total_mb} | 内存问题监控 |

#### 8.2.3 错误事件

| 事件名称 | 触发时机 | 参数 | 用途 |
|---------|---------|------|------|
| error_occurred | 发生错误 | {error_code, error_message} | 错误率监控 |
| model_load_failed | 模型加载失败 | {model_name, error} | AI模型问题 |
| db_error | 数据库错误 | {operation, error} | 数据库问题 |
| crash | 应用崩溃 | {stack_trace} | 崩溃率监控 |

### 8.3 埋点示例

```typescript
// 搜索埋点
analytics.track('search_performed', {
  query_length: query.length,
  has_time_filter: !!timeRange,
  result_count: results.length,
  duration_ms: endTime - startTime,
});

// 扫描埋点
analytics.track('scan_completed', {
  total_photos: progress.total,
  indexed_count: progress.indexed,
  failed_count: progress.failed,
  duration_seconds: duration,
});

// 错误埋点
analytics.track('error_occurred', {
  error_code: error.code,
  error_type: error.type,
  screen: currentScreen,
});
```

| 指标名称 | 定义 | 目标值 |

#### 8.4.2 功能指标

| 指标名称 | 定义 | 目标值 |
|---------|------|--------|

| 扫描完成率 | 开始扫描的用户中完成比例 | > 80% |

| 指标名称 | 定义 | 目标值 |

| 崩溃率 | 崩溃次数 / 启动次数 | < 0.1% |

### 9.2 核心用户故事

#### US-001: 基本自然语言搜索
**作为** U1摄影爱好者，  
**我想要** 通过输入自然语言描述来搜索照片，  
**以便** 快速找到特定场景的照片而不需要手动浏览数千张照片。

**验收标准**:
- [ ] 可以输入"海边的日落"并找到匹配照片
- [ ] 可以输入"穿红衣服的人在公园"并找到匹配照片
- [ ] 搜索响应时间 < 500ms
- [ ] 支持中文输入

#### US-002: 找特定时间的照片
**作为** U2宝妈，  
**我想要** 搜索"去年春节全家福"，  
**以便** 找到去年春节期间拍摄的家庭照片。

**验收标准**:
- [ ] 支持"去年"、"上个月"、"上周"等相对时间
- [ ] 支持"2024年春节"等具体时间段
- [ ] 时间和场景可以组合搜索

#### US-003: 查看扫描进度
**作为** U1摄影爱好者（拥有1万+照片），  
**我想要** 看到详细的扫描进度，  
**以便** 了解索引何时完成。

**验收标准**:
- [ ] 显示已扫描/总照片数
- [ ] 显示预计剩余时间
- [ ] 支持暂停/恢复扫描

#### US-004: 完全本地处理
**作为** U4隐私敏感用户，  
**我想要** 所有AI处理和照片数据完全在本地，  
**以便** 我的照片不会上传到任何云端服务器。

**验收标准**:
- [ ] 无需网络权限即可使用核心功能
- [ ] 无需用户登录
- [ ] 应用不发送照片到外部服务器
